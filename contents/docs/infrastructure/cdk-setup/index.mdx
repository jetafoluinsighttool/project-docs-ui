# 🚀 Configuración y Despliegue con AWS CDK

Esta sección cubre el proceso de configuración y despliegue de la infraestructura utilizando **AWS Cloud Development Kit (CDK)**. AWS CDK permite definir y gestionar infraestructura en la nube mediante código, proporcionando flexibilidad y reutilización en los despliegues.

## 🛠 ¿Qué es AWS CDK?

AWS CDK es un framework de infraestructura como código (IaC) que permite definir recursos de AWS utilizando lenguajes de programación como **TypeScript, JavaScript, Python y más**. En este proyecto, se usa **TypeScript** para definir las pilas (**stacks**) que despliegan servicios como **Lambdas, API Gateways, bases de datos DynamoDB y Cognito**.

## 📝 Comandos para Desplegar

Para realizar despliegues con CDK, se utilizan los siguientes comandos básicos:

### 🔹 Inicializar el Proyecto

Si es la primera vez que se usa CDK en la cuenta de AWS, se debe ejecutar:

```sh
cdk bootstrap
```

Este comando configura la cuenta para utilizar CDK y crea los recursos necesarios para el despliegue.

### 🔹 Revisar Cambios Antes del Despliegue

Antes de desplegar, se recomienda revisar los cambios que se aplicarán:

```sh
cdk diff
```

Este comando muestra una comparación entre la infraestructura actual y la que se va a desplegar.

### 🔹 Desplegar un Entorno Específico

Para desplegar en el entorno de pruebas (`staging`):

```sh
cdk deploy ProjectInstanceApiStack-Staging
```

Para desplegar en producción (`production`):

```sh
cdk deploy ProjectInstanceApiStack-Production
```

Si se requiere desplegar todas las pilas (`stacks`) al mismo tiempo:

```sh
cdk deploy --all
```

### 🔹 Eliminar un Despliegue

Si es necesario eliminar recursos, se puede hacer con:

```sh
cdk destroy ProjectInstanceApiStack-Staging
```

O eliminar todas las pilas con:

```sh
cdk destroy --all
```

---

💡 **Recomendación:** Siempre verificar el entorno antes de ejecutar cualquier comando de despliegue. En producción, un error puede afectar los datos en uso.
