# 🖥️ Interfaz Principal – JET AFOLU Insight Tool

La interfaz principal de **JET AFOLU Insight Tool** está diseñada para ofrecer una experiencia de usuario eficiente y adaptable, basada en **Next.js** y un ecosistema moderno de tecnologías frontend y backend.

## 🚀 Tecnologías utilizadas

La aplicación está construida con un stack moderno para garantizar rendimiento, seguridad y escalabilidad:

### 🏗️ **Framework y Lenguajes**

- **⚡ Next.js:** [nextjs.org](https://nextjs.org/) – Framework para SSR, SSG y optimización de rendimiento.
- **📌 TypeScript:** [typescriptlang.org](https://www.typescriptlang.org/) – Tipado estático para mayor seguridad en el código.
- **⚛️ React:** [react.dev](https://es.react.dev/) – Librería de componentes reutilizables y eficiente.

### 🎨 **Estilos y Componentes UI**

- **🎨 TailwindCSS:** [tailwindcss.com](https://tailwindcss.com/) – Estilización rápida y adaptable con clases utilitarias.
- **💅 Emotion:** [emotion.sh](https://emotion.sh/docs/introduction) – Librería de estilos en línea con soporte para temas.
- **🖌️ ShadCN UI:** [ui.shadcn.com](https://ui.shadcn.com/) – Librería de componentes accesibles y modernos.
- **🖍️ Radix UI:** [radix-ui.com](https://www.radix-ui.com/) – Componentes accesibles para diálogos, menús, tooltips y más.

### 📡 **Gestión de Estado y Datos**

- **📦 Zustand:** [zustand-demo.pmnd.rs](https://zustand-demo.pmnd.rs/) – Gestión de estado ligera y eficiente.
- **🔄 SWR:** [swr.vercel.app](https://swr.vercel.app/) – Manejo de datos remotos con revalidación automática.
- **📊 React Table:** [tanstack.com/table](https://tanstack.com/table) – Gestión y renderizado de tablas con gran flexibilidad.

### 🔐 **Autenticación y Seguridad**

- **🛡️ AWS Amplify:** [aws.amazon.com/amplify](https://aws.amazon.com/amplify/) – Manejo de autenticación y autorización con Cognito.
- **🔑 AWS Cognito SDK:** [npm aws-sdk](https://www.npmjs.com/package/@aws-sdk/client-cognito-identity-provider) – Interacción directa con AWS Cognito para autenticación segura.

## 🌍 Conexión con el Backend

La aplicación consume datos y funcionalidades de diversas APIs backend que manejan la lógica del sistema. La comunicación se realiza a través de **AWS API Gateway** y **Lambda**, con autenticación vía **Cognito**.

📡 **Principales APIs conectadas:**

- **🛠️ `project-instance-api`** → Consulta y envío de respuestas de usuarios.
- **🗂️ `project-template-api`** → Consulta de preguntas en múltiples idiomas desde DynamoDB.
- **📊 `project-reviewer-api`** → Generación de métricas y análisis de respuestas.
- **🔐 `project-auth-api`** → Gestión de autenticación y autorización.

La aplicación utiliza **Axios** y **SWR** para manejar las peticiones HTTP de forma eficiente.

## 🏗️ Estructura de Layouts y Perfiles

La interfaz está diseñada para adaptarse a diferentes tipos de usuarios dentro de la plataforma. Dependiendo del perfil del usuario, la aplicación carga distintos layouts y permisos.

👤 **Tipos de Perfiles:**

- **Administrador:** Acceso total, gestión de usuarios y configuración del sistema.
- **Revisor:** Puede analizar respuestas y generar métricas.
- **Contribuidor:** Envía respuestas y consulta preguntas dentro del sistema.

📌 **La interfaz se adapta dinámicamente según los permisos del usuario, asegurando una experiencia personalizada y segura.**
