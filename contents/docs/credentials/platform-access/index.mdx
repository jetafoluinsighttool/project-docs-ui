Para acceder a los servicios principales del proyecto (**AWS, GitHub y Vercel**), sigue los pasos descritos a continuación. Las credenciales se encuentran almacenadas en un documento compartido.

## 📌 Acceder a las Credenciales

Las credenciales necesarias para iniciar sesión en **AWS y GitHub** se encuentran en el siguiente enlace:

🔗 **[Google Spreadsheet - Credenciales](https://docs.google.com/spreadsheets/d/15XHqiMEk29kPfDfd6_N8gaF-sPJdY3knEDVJeXxIVik/edit?gid=0#gid=0)**

Asegúrate de contar con acceso al documento para poder visualizar los datos.

## 🌐 Inicio de Sesión en los Servicios

### 1️⃣ AWS

Para iniciar sesión en **AWS**, sigue estos pasos:

- Dirígete a [AWS Console](https://aws.amazon.com/console/).
- Usa las credenciales almacenadas en el documento.
- Verifica el acceso y la región correcta antes de realizar despliegues.

### 2️⃣ GitHub

Para acceder a **GitHub**:

- Ve a [GitHub](https://github.com/).
- Inicia sesión con las credenciales disponibles en el documento.
- GitHub es necesario para gestionar el código y realizar despliegues.

### 3️⃣ Vercel

Para gestionar despliegues en **Vercel**, accede con tu cuenta de **GitHub**:

- Dirígete a [Vercel](https://vercel.com/).
- Inicia sesión utilizando tu cuenta de GitHub.
- Desde Vercel, puedes visualizar y gestionar los despliegues en la plataforma.

---

💡 **Recomendación:** Mantén las credenciales seguras y no compartas accesos sin autorización. Asegúrate de cerrar sesión en dispositivos públicos para evitar accesos no deseados.
